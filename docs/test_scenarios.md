# Тестирование пользовательских сценариев

---

## Сценарий 1: Поиск негативных комментариев

**Цель:**  
Найти негативные комментарии от пользователя

**Шаги:**
1. Открыть страницу
2. Перейти во вкладку "Комментарии"
3. Открыть поиск
4. Ввести ключевое слово (например, "негатив")
5. Применить фильтры при необходимости

**Ожидаемый результат:**
- Отображаются релевантные комментарии

**Проверки:**
- Поиск учитывает введённое слово
- Фильтры корректно применяются
- Результаты обновляются

**Риски:**
- Поиск не учитывает регистр
- Не работает частичное совпадение
- Нет сообщения при пустом результате

---

## Сценарий 2: Редактирование и подтверждение ответа

**Цель:**  
Изменить и подтвердить ответ

**Шаги:**
1. Найти комментарий
2. Нажать "Редактировать"
3. Изменить текст
4. Сохранить изменения
5. Подтвердить ответ

**Ожидаемый результат:**
- Текст обновляется
- Статус ответа изменяется

**Проверки:**
- Появляется поле редактирования
- Изменения сохраняются
- После refresh данные не теряются

**Риски:**
- Потеря данных при переключении вкладок
- Отсутствие подтверждения сохранения
- Некорректная работа кнопки "Отмена"

---

## Сценарий 3: Экстренная остановка ботов

**Цель:**  
Остановить работу всех ботов

**Шаги:**
1. Найти тумблер управления ботами
2. Переключить его в состояние "выключено"

**Ожидаемый результат:**
- Боты останавливаются
- UI обновляет статус

**Проверки:**
- Отображается уведомление
- API возвращает корректный статус
- UI синхронизирован с backend

**Риски:**
- Нет подтверждения действия
- UI не соответствует backend состоянию
- Ошибка при сетевом запросе

---

## Дополнительные проверки

### Data validation:
- Даты отображаются в корректном формате
- Сортировка работает корректно
- Числовые значения валидны (нет NaN/null)

### Edge cases:
- Ввод пустого текста → ошибка
- Отмена редактирования → восстановление текста
- Проверка сортировки (новые/старые)
- Проверка обновления счетчиков
