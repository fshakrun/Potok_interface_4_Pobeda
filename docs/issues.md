# Поиск проблем в интерфейсе

## 1. Отсутствие явного состояния активной вкладки

**Местоположение:** Навигационные вкладки (Все / Комментарии / Отзывы)

**Описание:**  
После клика по вкладке визуально не всегда очевидно, какая вкладка активна (класс `active` не меняется).

**Влияние:**  
Пользователь может потерять контекст и не понимать, какие данные сейчас отображаются.

**Рекомендация:**  
Добавить явное визуальное выделение активной вкладки (цвет, underline, фон).

---

## 2. Фильтры не имеют явной обратной связи

**Местоположение:** Кнопки фильтрации платформ (ВКонтакте и др.)

**Описание:**  
После нажатия на фильтр не происходит визуального изменения состояния кнопки.

**Влияние:**  
Пользователь не понимает, применился ли фильтр.

**Рекомендация:**  
Добавить состояние active/selected + возможность сброса фильтров.

---

## 3. Отсутствие элементов управления ботами

**Местоположение:** Ожидаемый блок управления ботами

**Описание:**  
В HTML отсутствует тумблер включения/выключения ботов.

**Влияние:**  
Невозможно протестировать критически важный функционал (экстренная остановка).

**Рекомендация:**  
Добавить UI-элемент управления и статус ботов.

## 4. Отсутствие CSS

**Местоположение:** Весь интерфейс

**Описание:**  
В предоставленном HTML отсутствуют стили (CSS), из-за чего интерфейс отображается некорректно.

**Влияние:**  
Невозможно полноценно оценить UX и визуальные состояния элементов (active, hover, ошибки).

**Рекомендация:**  
Подключить CSS или предоставить макет/дизайн.

---

## 5. Отсутствие валидации пустого ответа

**Местоположение:** Поле редактирования ответа

**Описание:**  
При сохранении пустого ответа не отображается ошибка.

**Влияние:**  
В систему могут попадать некорректные данные.

**Рекомендация:**  
Добавить обязательную валидацию и сообщение об ошибке.

---

## 6. Некорректная логика отмены редактирования

**Местоположение:** Поле редактирования ответа

**Описание:**  
После очистки текста и нажатия "Отмена" исходный текст не восстанавливается.

**Влияние:**  
Потеря данных и ухудшение UX.

**Рекомендация:**  
Хранить исходное значение и восстанавливать его при отмене.

## 7. Риск некорректного формата даты и чисел

**Местоположение:** Список комментариев, счетчики, даты

**Описание:**  
В интерфейсе отображаются даты и числовые значения (счетчики). При отсутствии валидации возможны:

- некорректный формат даты
- несортируемые значения
- некорректные числа (NaN, null)

**Влияние:**  
- неправильная сортировка (“сначала новые/старые”)
- потеря доверия пользователя
- ошибки отображения данных

**Рекомендация:**  
- валидировать формат даты на backend
- использовать единый формат (например ISO → UI формат)
- валидировать числовые значения
